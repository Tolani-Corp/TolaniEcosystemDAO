               SECTION 3: SMART CONTRACT ARCHITECTURE
================================================================================

3.1 - Core Governance Contracts

TOLANIGOVERNOR (OpenZeppelin Governor)
- Purpose: Manages on-chain proposal lifecycle and voting
- Standards: OpenZeppelin Governor with extensions
- Key Functions:
  * propose() - Submit governance proposals
  * castVote() - Cast vote on active proposals
  * execute() - Execute approved proposals
  * queue() - Queue proposals for timelock execution

TOLANITIMELOCK (TimelockController)
- Purpose: Enforces delay between approval and execution
- Delay Period: 48 hours (172,800 seconds)
- Key Functions:
  * schedule() - Schedule operation for execution
  * execute() - Execute after delay
  * cancel() - Cancel scheduled operation (admin only)

3.2 - Token Contracts

TOLANITOKEN (TUT)
- Pattern: UUPS Upgradeable Proxy
- Standards: ERC-20, ERC-20Votes, ERC-20Permit
- Key Functions:
  * transfer() - Standard ERC-20 transfer
  * delegate() - Delegate voting power
  * getPastVotes() - Historical vote checkpoints

UTUTSIMPLE (uTUT)
- Pattern: Non-upgradeable
- Standards: ERC-20 (Non-transferable)
- Key Functions:
  * mint() - Award training rewards (admin only)
  * burn() - Redeem tokens (admin only)

3.3 - Treasury Contracts

TOLANITREASURY
- Purpose: Manages DAO on-chain treasury
- Controls: Multi-signature + Governance
- Key Functions:
  * executeTransaction() - Execute approved transfers
  * setAllocation() - Update allocation categories

GNOSIS SAFE
- Address: 0xa56eb5E3990C740C8c58F02eAD263feF02567677
- Threshold: 3-of-5 signatures required
- Purpose: Emergency controls and off-chain asset management

3.4 - DeFi Contracts

TOLANISTAKING
- Purpose: TUT staking for rewards
- Key Functions:
  * stake() - Deposit TUT tokens
  * unstake() - Withdraw staked tokens
  * claimRewards() - Claim accumulated rewards

TOLANILPSTAKING
- Purpose: Liquidity provider incentives
- Integration: Uniswap V3 LP positions
- Key Functions:
  * depositLP() - Stake LP tokens
  * withdrawLP() - Unstake LP tokens

4.2 - Proposal Types

STANDARD PROPOSAL:
- Changes to protocol parameters
- Treasury disbursements under $100,000
- Contract upgrades (with audit)
- Policy amendments

CRITICAL PROPOSAL:
- Changes requiring >$100,000 treasury movement
- Multi-sig threshold changes
- Protocol emergency pauses
- Additional 7-day community review recommended

4.3 - Emergency Powers

The Gnosis Safe multi-sig retains emergency powers:
- Pause protocols in case of exploit (2-of-5 required)
- Execute emergency upgrades (3-of-5 required)
- All emergency actions subject to post-hoc governance review

5.1 - Security Measures

(a) AUDIT STATUS:
    - Core contracts: [Audit pending - to be completed before mainnet]
    - OpenZeppelin dependencies: Audited by OpenZeppelin
    - Proxy pattern: Standard UUPS (audited)

(c) UPGRADE MECHANISMS:
    - UUPS proxy pattern for TUT token
    - Governance-controlled upgrade process
    - Implementation address verified on-chain

5.2 - Known Risks and Limitations

SMART CONTRACT RISKS:
(a) Code bugs: Despite audits, undiscovered bugs may exist
(b) Upgrade risks: Future upgrades may introduce vulnerabilities
(c) Dependency risks: OpenZeppelin or other dependencies may have issues
(d) Oracle risks: External data dependencies may be manipulated

GOVERNANCE RISKS:
(a) Voter apathy: Low participation may enable minority control
(b) Vote buying: Token accumulation may concentrate power
(c) Proposal spam: Malicious proposals may require filtering
(d) Flash loan attacks: Vote manipulation via borrowed tokens

ECONOMIC RISKS:
(a) Token price volatility: Market forces may impact value
(b) Liquidity risks: Insufficient liquidity may affect trading
(c) Impermanent loss: LP providers may experience losses
(d) Smart contract exploits: Funds may be lost to attacks

5.3 - Bug Bounty Program

The DAO maintains a bug bounty program for responsible disclosure:
- Critical vulnerabilities: Up to $50,000 TUT
- High severity: Up to $25,000 TUT
- Medium severity: Up to $10,000 TUT
- Low severity: Up to $2,500 TUT

Report vulnerabilities to: security@tolani.dao
